# MiniSQL — Custom SQL Database Engine
## Project Roadmap with Timeline

========================================
PHASE 1 — CORE STORAGE ENGINE (Weeks 1–2)
Goal: Create a table file, insert rows, and read them back.
Tasks:
  1.1 Understand that one table = one file on disk
  1.2 Define fixed row structure (column order and types)
  1.3 Choose storage format (text-based or binary)
  1.4 Design serialization/deserialization logic conceptually
  1.5 Define Insert = append row to end of file
  1.6 Define Select = read file line by line
Milestone Output: Database can create a table, insert rows, and retrieve them from disk.

========================================
PHASE 2 — SCHEMA & CATALOG MANAGER (Weeks 3–4)
Goal: Support multiple tables and store their schema persistently.
Tasks:
  2.1 Decide on a Catalog File to store metadata
  2.2 Decide Schema format: Table Name + Columns + Types
  2.3 On CREATE TABLE, write schema info into catalog
  2.4 On INSERT/SELECT, validate data structure using schema from catalog
Milestone Output: Database remembers tables and schemas after program restart.

========================================
PHASE 3 — MINIMAL SQL PARSER (Weeks 5–6)
Goal: Accept SQL-like text commands and route them to execution logic.
Tasks:
  3.1 Distinguish between raw command string and execution operation
  3.2 Conceptualize Tokenizer (split command into pieces)
  3.3 Command Router: match commands to functions
  3.4 Support CREATE TABLE, INSERT INTO, SELECT *
Milestone Output: User can type SQL-like commands and database responds correctly.

========================================
PHASE 4 — WHERE CLAUSE FILTERING (Weeks 7–8)
Goal: Filter results based on conditions in SELECT.
Tasks:
  4.1 Extend parser to detect WHERE clause
  4.2 Implement basic condition operators (=, <, >)
  4.3 Apply filters during SELECT scan
Milestone Output: SELECT queries with WHERE return filtered results.

========================================
PHASE 5 — INDEXING SYSTEM (Weeks 9–10)
Goal: Accelerate SELECT WHERE queries using indexes.
Tasks:
  5.1 Choose index structure (B+ Tree or Hash Map)
  5.2 Build index for specific column
  5.3 Update index whenever INSERT happens
  5.4 Use index instead of scanning whole file
Milestone Output: Queries with WHERE are significantly faster.

========================================
PHASE 6 — WAL & RECOVERY (Weeks 11–12)
Goal: Prevent data corruption and allow crash recovery.
Tasks:
  6.1 Design Write Ahead Log Format
  6.2 Log all operations before applying to table files
  6.3 Implement commit markers
  6.4 On restart, replay incomplete logs to restore consistency
Milestone Output: Database recovers safely after unexpected shutdown.

========================================
OPTIONAL PHASE 7 — CONCURRENCY CONTROL (After Month 3)
Goal: Allow safe multi-user/multi-thread access.
Tasks:
  7.1 Understand read vs write conflict
  7.2 Add table/row locking or MVCC versioning
  7.3 Deadlock detection strategy
Milestone Output: Database supports concurrent operations safely.

========================================
OPTIONAL PHASE 8 — QUERY OPTIMIZER (After Month 3)
Goal: Automatically choose best execution strategy.
Tasks:
  8.1 Collect table/index statistics
  8.2 Estimate cost for scan vs index lookup
  8.3 Create execution plan selector
Milestone Output: Database intelligently chooses optimal execution plan.

========================================
PHASE 9 — CLI & DOCUMENTATION (Ongoing)
Goal: Provide an interactive shell and proper documentation.
Tasks:
  9.1 Implement REPL (Read-Eval-Print Loop)
  9.2 Format output neatly
  9.3 Write README and diagrams explaining architecture
Milestone Output: Database feels like a real SQL engine with user documentation.

========================================
TOTAL TIMELINE SUMMARY:
- Weeks 1–2: PHASE 1 (Core Storage)
- Weeks 3–4: PHASE 2 (Schema & Catalog)
- Weeks 5–6: PHASE 3 (SQL Parser)
- Weeks 7–8: PHASE 4 (WHERE Filtering)
- Weeks 9–10: PHASE 5 (Indexing)
- Weeks 11–12: PHASE 6 (WAL & Recovery)
- After Month 3: Optional Phase 7 & 8 (Concurrency, Optimizer)
- Ongoing: Phase 9 (CLI & Documentation)

End of File.
